1

Placer les équipements dans Packet Tracer

Interface · Workspace · Glisser-déposer

Découverte
"Tu lances Packet Tracer pour la première fois. La grande zone blanche est ton espace de travail. En bas, une palette d'équipements attend — switches, routeurs, PC. Ta mission : construire le réseau du schéma de zéro."
  • 1
    Lance Cisco Packet Tracer. La grande zone blanche centrale est ton espace de travail.
  • 2
    En bas à gauche, clique sur End Devices (icône PC). Glisse 2 PC génériques dans la zone de travail.
  • 3
    Clique sur Network Devices → Switches. Place un Cisco 2960 au centre.
  • 4
    Clique sur Network Devices → Routers. Place un Cisco 2811 en dessous du switch.
  • 5
    Double-clique sur chaque PC → onglet Config → champ Display Name : renomme-les PC-Fatima et PC-Karim.
💡Utilise Ctrl+Z pour annuler. Tu peux déplacer les équipements à tout moment en les glissant.

Où se trouve la palette d'équipements dans Packet Tracer ?

2

Câbler les équipements

Câbles droits · Câble croisé · Voyants de statut

Câblage
"Les équipements sont posés mais ne communiquent pas encore. Dans un vrai datacenter on branche des câbles RJ45 — ici deux clics suffisent. Le bon type de câble est crucial !"
  • 1
    En bas, clique sur Connections (icône éclair). Sélectionne Copper Straight-Through (câble droit — ligne noire pleine).
  • 2
    Relie PC-Fatima (FastEthernet0) au Switch (FastEthernet0/1).
  • 3
    Répète : relie PC-Karim (FastEthernet0) au Switch (FastEthernet0/2).
  • 4
    Change de câble : Copper Cross-Over (câble croisé, tirets). Relie le Switch (Fa0/3) → Routeur (FastEthernet0/0).
  • 5
    Les voyants passent au vert = connexion OK. Orange = en cours, attends quelques secondes.
💡Mémo : équipements différents → câble droit · équipements identiques → câble croisé.

3

Configurer les adresses IP des PC

IP statique · Masque de sous-réseau · Passerelle

Config IP
"Les câbles sont branchés. Mais sans adresse IP, les paquets ne savent pas où aller — comme une enveloppe sans adresse. Donnons une identité réseau à chaque PC."
  • 1
    Double-clique sur PC-Fatima → onglet Desktop → IP Configuration → coche Static.
  • 2
    IP Address : 192.168.1.10 · Subnet Mask : 255.255.255.0 · Default Gateway : 192.168.1.1
  • 3
    Ferme. Double-clique sur PC-Karim → même chemin → coche Static.
  • 4
    IP Address : 192.168.1.20 · Subnet Mask : 255.255.255.0 · Default Gateway : 192.168.1.1
Vérification · Command PromptC:\> ipconfig

# Résultat attendu (PC-Fatima) :
IPv4 Address. . .: 192.168.1.10
Subnet Mask . . .: 255.255.255.0
Default Gateway .: 192.168.1.1

4

Configurer le routeur en ligne de commande

Cisco IOS · enable · no shutdown

CLI Cisco
"Les PC sont configurés. Maintenant le routeur — la porte de sortie du réseau local. Il se configure en ligne de commande avec Cisco IOS. Bienvenue dans la vraie vie réseau !"
  • 1
    Double-clique sur le Routeur 2811 → onglet CLI.
  • 2
    Appuie sur Entrée. Le prompt Router> apparaît.
  • 3
    Tape les commandes ci-dessous dans l'ordre, Entrée après chacune.
  • 4
    Après no shutdown, le message FastEthernet0/0, changed state to up confirme que c'est actif.
Routeur 2811 · CLIRouter> enable
Router# configure terminal
! Mode configuration globale
Router(config)# interface FastEthernet0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
! Active l'interface
Router(config-if)# exit
Router(config)# exit
Router# write memory
VérificationRouter# show ip interface brief

# Résultat attendu :
FastEthernet0/0 192.168.1.1 YES up up

5

Tester la connectivité avec ping

Validation · Dépannage · Mode simulation

Validation
"Le réseau est construit. L'heure de vérité : PC-Fatima peut-elle parler à PC-Karim ? On lance les pings — le test ultime de la connectivité réseau !"
  • 1
    PC-Fatima → Desktop → Command Prompt. Teste vers PC-Karim :
Test PC ↔ PCC:\> ping 192.168.1.20

# 4 réponses attendues :
Reply from 192.168.1.20: bytes=32 time<1ms TTL=128
Reply from 192.168.1.20: bytes=32 time<1ms TTL=128
  • 2
    Teste vers la passerelle (le routeur) :
Test PC → RouteurC:\> ping 192.168.1.1

Reply from 192.168.1.1: bytes=32 time=1ms TTL=255
  • 3
    Active le mode Simulation (horloge en bas à droite) et relance un ping — vois les paquets voyager visuellement !
  • 4
    Si Request timed out : vérifie IPs, câblage, et show ip interface brief sur le routeur.
🔧Dépannage : voyants câbles verts ? IPs correctes sur les PC ? Routeur affiche "up up" ?
